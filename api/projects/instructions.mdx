## Overview

Instructions are the entry points to your Solana program. These endpoints allow you to list all instructions and retrieve detailed information about specific instructions, including their accounts and arguments.

## List All Instructions

<CodeGroup>
```bash Endpoint
GET /api/:projectId/instructions
```
</CodeGroup>

Retrieve a complete list of all instructions defined in the program's IDL.

### Path Parameters

<ParamField path="projectId" type="string" required>
  The unique identifier of the project
</ParamField>

### Response

<ResponseField name="projectId" type="string">
  The project identifier
</ResponseField>

<ResponseField name="programName" type="string">
  Name of the Anchor program from the IDL
</ResponseField>

<ResponseField name="programId" type="string">
  Solana program ID (base58 encoded)
</ResponseField>

<ResponseField name="instructions" type="array">
  List of instruction objects
  
  <Expandable title="Instruction Object">
    <ResponseField name="name" type="string">
      Instruction name (camelCase)
    </ResponseField>
    
    <ResponseField name="docs" type="array">
      Documentation strings from the IDL
    </ResponseField>
    
    <ResponseField name="accountCount" type="integer">
      Number of accounts required by this instruction
    </ResponseField>
    
    <ResponseField name="argCount" type="integer">
      Number of arguments required by this instruction
    </ResponseField>
    
    <ResponseField name="accounts" type="array">
      Account metadata
      
      <Expandable title="Account Meta">
        <ResponseField name="name" type="string">
          Account name
        </ResponseField>
        
        <ResponseField name="isMut" type="boolean">
          Whether the account is mutable
        </ResponseField>
        
        <ResponseField name="isSigner" type="boolean">
          Whether the account must sign
        </ResponseField>
        
        <ResponseField name="isOptional" type="boolean">
          Whether the account is optional
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="args" type="array">
      Instruction arguments
      
      <Expandable title="Argument">
        <ResponseField name="name" type="string">
          Argument name
        </ResponseField>
        
        <ResponseField name="type" type="string">
          Type representation (e.g., "u64", "string", object with amount field)
        </ResponseField>
        
        <ResponseField name="isDefinedType" type="boolean">
          Whether this references a custom type
        </ResponseField>
        
        <ResponseField name="definedTypeName" type="string">
          Name of the custom type (if applicable)
        </ResponseField>
        
        <ResponseField name="fields" type="array">
          Nested fields for struct types
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Get Instruction Details

<CodeGroup>
```bash Endpoint
GET /api/:projectId/instructions/:name
```
</CodeGroup>

Retrieve detailed information about a specific instruction, including default values and PDA seed information.

### Path Parameters

<ParamField path="projectId" type="string" required>
  The unique identifier of the project
</ParamField>

<ParamField path="name" type="string" required>
  The instruction name (camelCase as defined in the IDL)
</ParamField>

### Response

<ResponseField name="projectId" type="string">
  The project identifier
</ResponseField>

<ResponseField name="programId" type="string">
  Solana program ID
</ResponseField>

<ResponseField name="instruction" type="object">
  Detailed instruction information
  
  <Expandable title="Instruction Details">
    <ResponseField name="name" type="string">
      Instruction name
    </ResponseField>
    
    <ResponseField name="docs" type="array">
      Documentation strings from the IDL
    </ResponseField>
    
    <ResponseField name="accounts" type="array">
      Account requirements with PDA information
      
      <Expandable title="Account">
        <ResponseField name="name" type="string">
          Account name
        </ResponseField>
        
        <ResponseField name="isMut" type="boolean">
          Whether the account is mutable
        </ResponseField>
        
        <ResponseField name="isSigner" type="boolean">
          Whether the account must sign
        </ResponseField>
        
        <ResponseField name="isOptional" type="boolean">
          Whether the account is optional
        </ResponseField>
        
        <ResponseField name="pda" type="object">
          PDA seed information (if applicable)
          
          <Expandable title="PDA Seeds">
            <ResponseField name="seeds" type="array">
              List of seed definitions for deriving the PDA
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="args" type="array">
      Instruction arguments with default values
      
      <Expandable title="Argument">
        <ResponseField name="name" type="string">
          Argument name
        </ResponseField>
        
        <ResponseField name="type" type="string">
          Type representation
        </ResponseField>
        
        <ResponseField name="defaultValue" type="any">
          Default value for this type (useful for testing)
        </ResponseField>
        
        <ResponseField name="isDefinedType" type="boolean">
          Whether this references a custom type
        </ResponseField>
        
        <ResponseField name="definedTypeName" type="string">
          Name of the custom type (if applicable)
        </ResponseField>
        
        <ResponseField name="fields" type="array">
          Nested fields for struct types
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<AccordionGroup>
  <Accordion title="404 Not Found - Project">
    ```json
    {
      "error": "Project not found or not public"
    }
    ```
    
    The specified project ID does not exist, is private, or has no IDL uploaded.
  </Accordion>
  
  <Accordion title="404 Not Found - Instruction">
    ```json
    {
      "error": "Instruction \"invalidName\" not found"
    }
    ```
    
    The specified instruction name does not exist in the program's IDL.
  </Accordion>
  
  <Accordion title="500 Internal Server Error">
    ```json
    {
      "error": "Failed to list instructions"
    }
    ```
    
    The server encountered an error processing the request.
  </Accordion>
</AccordionGroup>

## Notes

<Info>
  **No authentication required** - These endpoints are public for all public projects.
</Info>

<Tip>
  Use the instruction details endpoint to get `defaultValue` for each argument, which is helpful for testing and building example transactions.
</Tip>

<Note>
  The `pda` field in account metadata indicates which accounts are Program Derived Addresses and provides the seed definitions needed to derive them.
</Note>
